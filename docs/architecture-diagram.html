④ Drizzle ORM とは何か
🗒️ SQL（直接書く場合）
SELECT * FROM sales_records
WHERE LEFT(date, 7) = '2026-03';

DELETE FROM shift_rows
WHERE month = '2026-03';

INSERT INTO sales_records
(date, staff_name, mnp_h, ...)
VALUES ('2026-03-01', '山田', '2', ...);
→ Drizzle に変換
💙 TypeScript（Drizzle を使う場合）
db.select().from(salesRecords)
  .where(eq(salesRecords.month, '2026-03'));

db.delete(shiftRows)
  .where(eq(shiftRows.month, '2026-03'));

db.insert(salesRecords)
  .values({ date: '2026-03-01',
    staffName: '山田', ... });

やっていることは同じ。Drizzle はタイプミスや型エラーをコード実行前に検出できる。

⑤ Neon vs Supabase
⚡
Neon
Serverless PostgreSQL
  • 🎯 DBに特化したシンプルなサービス
  • ▲ Vercel との統合が最も簡単
  • 🔌 独自HTTPプロトコルで高速接続
  • 💤 使わないと自動でスリープ（無料）
  • 🔐 認証機能はない（自前で実装）
✅ 本番運用中（メインDB）
🟢
Supabase
Open Source Firebase Alternative
  • 🔐 認証（Auth）が内蔵 ← 強み
  • ⚡ リアルタイム更新が内蔵
  • 🗂️ ファイルストレージが内蔵
  • 🌐 サーバーが東京（ap-northeast-1）
  • 🔌 標準TCPで接続（postgres.js）
📚 学習用・並行運用中

200ms で表示
スプレッドシートはデータ入力のまま維持しつつ、表示速度を 3〜10秒 → 200ms に短縮した。
これが「スプレッドシートをバックエンドに使う業務アプリのDB移行」というアーキテクチャパターン。
